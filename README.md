# Tax Workbench (Tax/Accounting Management System)

본 프로젝트는 10만 건 이상의 데이터를 효율적으로 관리하기 위한 업무 관리 시스템입니다. 대용량 데이터 환경에서의 브라우저 렌더링 성능 최적화와 데이터 무결성 유지를 위한 아키텍처 설계에 중점을 두었습니다.

---

## 1. 빌드 및 실행 방법 (How to Run)

### 1.1 통합 실행 스크립트
제공된 셸 스크립트를 통해 백엔드와 프론트엔드를 동시에 실행할 수 있습니다.
```bash
# 기본 실행 (Port: 19100, Local Auth Mode)
./run-local.sh

# 초기 데이터(20만 건) 적재 모드 실행
./run-local.sh --seed-large
```

### 1.2 수동 실행 가이드
- **Backend (Spring Boot):** `backend` 디렉토리에서 `./gradlew bootRun` 실행 (JDK 21 필요)
- **Frontend (React):** `frontend` 디렉토리에서 `npm install && npm run dev` 실행

---

## 2. Architectural Rationale (아키텍처 설계 근거)

### 2.1 성능 최적화
- **DOM 가상화 그리드:** 수십만 건의 데이터 조회 시 브라우저 메모리 부하를 줄이기 위해 뷰포트 내 요소만 렌더링하도록 구현했습니다. 이를 통해 대량 데이터 환경에서도 안정적인 스크롤 성능을 유지합니다.
- **영속성 계층 분리 (CQRS):** 데이터 상태 변경(JPA)과 대용량 조회(MyBatis)의 책임을 분리했습니다. 이를 통해 조회 쿼리의 성능을 최적화하고, 복잡한 필터링 요구사항에 유연하게 대응하도록 설계했습니다.

### 2.2 데이터 신뢰성 및 가용성
- **비동기 작업 관리:** CSV 업로드 및 대규모 데이터 내보내기 등 시간이 소요되는 작업을 비동기로 처리합니다. '작업 관제' 메뉴를 통해 백그라운드 작업 상태를 실시간으로 확인할 수 있습니다.
- **낙관적 잠금 (Optimistic Locking):** JPA `@Version` 필드와 HTTP `If-Match` 헤더를 활용하여, 다중 사용자 수정 시 발생할 수 있는 데이터 덮어쓰기(Lost Update) 문제를 방지하도록 설계했습니다.

## 3. Trade-off Analysis (설계 타협점 분석)

- **커스텀 그리드 적용:** 기성 라이브러리 대신 커스텀 그리드를 구현하여 의존성을 줄이고, 클립보드 연동 및 인라인 편집 등 비즈니스에 특화된 기능을 직접 제어하도록 구성했습니다.
- **내장 스레드 풀 활용:** 인프라 복잡도를 낮추기 위해 외부 메시지 큐(MQ) 대신 내장 `ExecutorService`를 활용했습니다. 이에 따른 프로세스 종료 시 작업 유실 위험은 서버 부팅 시의 **자동 작업 복구 로직**을 통해 보완했습니다.

---

## 4. Cloud & PaaS Strategy (운영 및 확장 전략)

실제 프로덕션 환경에서 트래픽과 데이터가 증가할 경우 다음과 같은 PaaS 기반 확장 전략을 고려할 수 있습니다.

1.  **메시지 큐(MQ) 도입:** 내장 비동기 로직을 **Azure Service Bus** 또는 **AWS SQS**로 전환하여 작업의 영속성을 확보하고, API 서버와 워커(Worker) 노드를 독립적으로 확장(Scale-out)합니다.
2.  **분산 캐싱:** **Azure Cache for Redis**를 도입하여 분산 환경에서의 작업 상태 공유 및 반복적인 조회 요청에 대한 응답 속도를 개선합니다.
3.  **컴퓨팅 자원 분리:** 대규모 파일 생성이나 통계 계산 등 CPU 집약적인 로직을 **Azure Functions** 등의 서버리스 환경으로 분리하여 메인 서비스의 자원을 보호합니다.

## 5. AI Collaboration Reflection (AI 협업 회고)

본 프로젝트는 **Gemini CLI**와 **Claude 3.5** 등 AI 에이전트를 활용한 협업 방식으로 진행되었습니다.

- **AI 활용 사례:** 테마 시스템의 CSS 변수 체계 설계, 비동기 상태 관리 로직 구현, 반복적인 UI 스타일 수정 작업 등에 AI를 활용하여 생산성을 높였습니다.
- **한계 극복:** AI가 대규모 코드 수정 중 문맥을 잃거나 인터페이스 정합성이 어긋나는 경우, 개발자가 직접 아키텍처 가이드라인을 재설정하고 수동으로 코드를 검증/조정하여 시스템의 안정성을 확보했습니다.
